- Explorer structure d'un nouveau projet avant contribution

**Exemple de prompt explicite**:
```

all analytics tracking events in the app"
```

### 3. ⏮️ Checkpoints & Rewind - Undo Intelligent

**Définition**: Système de **sauvegarde automatique** qui capture l'état du code ET de la conversation avant chaque modification initiée par Claude. Permet de revenir en arrière instantanément avec la commande `/rewind`.

```
╔═══════════════════════════════════════════════════════╗
║           CHECKPOINTS - SYSTÈME DE SAUVEGARDE         ║
╚═══════════════════════════════════════════════════════╝

                    TIMELINE

Checkpoint 1         Checkpoint 2         Checkpoint 3
    ├───────────────────┼───────────────────┼──────> Time
    │                   │                   │
    │ "Add auth"        │ "Fix bug"         │ "Refactor"
    │                   │                   │
    ▼                   ▼                   ▼
┌─────────┐       ┌─────────┐       ┌─────────┐
│ Code v1 │       │ Code v2 │       │ Code v3 │
│ Chat v1 │       │ Chat v2 │       │ Chat v3 │
└─────────┘       └─────────┘       └─────────┘

          🔄 COMMANDE: /rewind (ou /re)

┌──────────────────────────────────────────────────────┐
│  SÉLECTIONNER CHECKPOINT                             │
│                                                      │
│  [ ] Checkpoint 3 - "Refactor" (5 min ago)          │
│  [ ] Checkpoint 2 - "Fix bug" (15 min ago)          │
│  [x] Checkpoint 1 - "Add auth" (30 min ago)         │
└──────────────────────────────────────────────────────┘
                        │
                        ▼
        ┌───────────────────────────────┐
        │  REWIND OPTIONS               │
        │                               │
        │  [x] Code                     │
        │  [x] Context (conversation)   │
        │  [ ] Both                     │
        └───────────────────────────────┘
                        │
                        ▼
            ✅ Restored to Checkpoint 1

⚠️  WORKFLOW: Linear Only (pas de back & forth)

    CP1 ──> CP2 ──> CP3
            ▲
            │
       Rewind ici
            │
    CP1 ──> CP2 ──> CP4 (nouveau)
            (point de non-retour à CP3)
```

**Avantages**:
- ✅ **Automatique** (1 checkpoint par prompt, zéro config)
- ✅ **Sauve code + conversation** (élimine contexte empoisonné)
- ✅ **Exploration sans risque** (test directions alternatives)
- ✅ **Complémentaire à Git** (undo session-level rapide)

**Limitations**:
- ❌ **Linéaire uniquement** (pas de jump back & forth entre checkpoints)
- ❌ Ne couvre **pas les changements manuels** (bash, edits directs)
- ❌ **Pas un remplacement de Git** (session-level seulement)

**Cas d'usage**:
- Claude part dans mauvaise direction → `/rewind` pour revenir
- Tester approche alternative sans perdre version actuelle
- Conversation context pollué par erreurs → rewind context uniquement
- Recovery rapide après série de mauvaises modifications

**Pro Tips**:
- Utiliser `/re` pour quick access
- Rewind code + context ensemble pour fresh start
- Toujours utiliser Git pour version control long-terme

### 4. 🎓 Skills - Connaissances Spécialisées Packagees

**Définition**: **Packages d'instructions réutilisables** pour enseigner à Claude votre façon de travailler. Un Skill = dossier + fichier `skill.md` contenant instructions, code, et ressources. Utilise **progressive disclosure** (Claude voit nom/description d'abord, charge le contenu full uniquement si pertinent).

```
╔═══════════════════════════════════════════════════════╗
║              SKILLS - ARCHITECTURE                    ║
╚═══════════════════════════════════════════════════════╝

📦 my-skill/
┣━━ 📄 skill.md ← Instructions principales
┣━━ 📄 examples.md
┣━━ 📄 templates/
┃   ┗━━ boilerplate.ts
┗━━ 📄 resources/
    ┗━━ references.md

╔════════════════════════════════════════════════════╗
║  PROGRESSIVE DISCLOSURE (Économie de Contexte)     ║
╚════════════════════════════════════════════════════╝

┌─────────────────────────────────────────┐
│  ÉTAPE 1: Claude voit liste skills      │
│                                         │
│  Available Skills:                      │
│  • pdf-processor: Process PDF docs      │
│  • test-driven: TDD framework           │
│  • meeting-insights: Analyze meetings   │
└─────────────────────────────────────────┘
                    │
        Task nécessite TDD ?
                    │
                    ▼ OUI
┌─────────────────────────────────────────┐
│  ÉTAPE 2: Load "test-driven" skill      │
│                                         │
│  [Full instructions loaded]             │
│  - Write test first                     │
│  - Implement minimal code               │
│  - Refactor                             │
└─────────────────────────────────────────┘

🌍 PORTABILITÉ UNIVERSELLE

┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ Claude Code  │    │  Claude App  │    │  Claude API  │
│              │    │              │    │              │
│  ~/skills/   │───>│  Same skills │───>│  Same skills │
└──────────────┘    └──────────────┘    └──────────────┘
                        ▲
                        │
                  Write once,
                  use everywhere

🎯 SKILLS PRÉ-CONSTRUITS ANTHROPIC

┌─────────────────────────────────────────────────────┐
│  📄 PDF Processor       📊 Excel Analyzer           │
│  📝 Word Editor         📈 PowerPoint Generator     │
│  🧪 Web App Testing     🔧 MCP Server Guide         │
│  🎯 Skill Creator       📚 EPUB Book Analyzer       │
└─────────────────────────────────────────────────────┘
```

**Avantages**:
- ✅ **Progressive disclosure** économise tokens (load uniquement si pertinent)
- ✅ **Portabilité totale** (Claude Code + App + API)
- ✅ **Réutilisabilité** (write once, use everywhere)
- ✅ **Skills pré-construits** Anthropic + communauté active

**Limitations**:
- ❌ Peut prendre beaucoup de contexte si skill volumineux chargé
- ❌ Nécessite bonne documentation pour être efficace
- ❌ Courbe d'apprentissage pour créer skills de qualité

**Cas d'usage**:
- Framework TDD personnalisé pour votre équipe
- Analyse de PDFs avec extraction structurée
- Testing automatisé d'apps web avec Playwright
- Création de MCPs servers avec best practices
- Meeting insights (transcription → action items)

**Exemple de Structure skill.md**:
```markdown
# Test-Driven Development Framework

## Description
Guide Claude through TDD workflow with red-green-refactor cycle.

## Instructions
1. Write failing test first
2. Implement minimal code to pass
3. Refactor while keeping tests green
4. Repeat

## Examples
[Code examples...]

## Resources
- Jest documentation
- Testing best practices
```

- ❌ **Vérification sécurité** (toujours vérifier source avant install)
- ❌ **Qualité variable** (communauté vs official plugins)

**Cas d'usage**:
